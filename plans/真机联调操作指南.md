# 学生端小程序真机联调操作指南

> 目标：让手机上的微信小程序能够访问本机运行的 Spring Boot 后端（localhost:8080）

---

## 前置检查

### 1. 确认 Java 版本（必须 ≥ 17）

```bash
java -version
mvn -version
```

- 如果 `java -version` 显示的版本低于 17，请安装 JDK 17 并配置 `JAVA_HOME` 环境变量。
- 如果 `mvn` 命令不存在，请安装 Apache Maven（建议 3.9+）。

---

## 第一步：启动后端服务

### 1.1 进入项目根目录

```bash
cd d:/-
```

> 以下所有命令均在此目录下执行。

### 1.2 使用 Maven 启动后端（mock 模式，无需数据库）

```bash
mvn spring-boot:run
```

或者先打包再运行（推荐，速度更快）：

```bash
mvn clean package -DskipTests
java -jar target/student-service-platform-backend-0.0.1-SNAPSHOT.jar
```

### 1.3 验证后端是否启动成功

在浏览器或 PowerShell 中访问：

```bash
curl http://localhost:8080/actuator/health
```

预期返回（JSON）：
```json
{"status":"UP"}
```

再测试登录接口：

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"2023100001","password":"123456"}'
```

预期返回包含 `token` 字段的 JSON。

> ⚠️ 常见坑：
> - 如果端口 8080 被占用，后端会启动失败。此时需要关闭占用 8080 的其他程序，或修改 `src/main/resources/application.yml` 中的 `server.port`。
> - 首次运行 Maven 可能会下载大量依赖，耗时较长，请保持网络畅通。

---

## 第二步：内网穿透（将 localhost:8080 暴露为公网 HTTPS）

由于微信小程序真机环境**只能访问 HTTPS 域名**（且需配置为合法域名），需要将本机的 `localhost:8080` 映射为一个公网可访问的 HTTPS 地址。

### 方案 A：使用 ngrok（推荐，简单免费）

#### 2A.1 下载 ngrok

访问 https://ngrok.com/download ，下载 Windows 版本并解压。

或直接用 PowerShell 下载（需管理员权限）：

```powershell
# 创建目录
mkdir C:\tools\ngrok -Force

# 下载（64位 Windows）
Invoke-WebRequest -Uri "https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-windows-amd64.zip" -OutFile "C:\tools\ngrok.zip"

# 解压
Expand-Archive -Path "C:\tools\ngrok.zip" -DestinationPath "C:\tools\ngrok" -Force
```

#### 2A.2 注册并获取 Authtoken

1. 访问 https://dashboard.ngrok.com/signup 注册账号
2. 登录后进入 https://dashboard.ngrok.com/get-started/your-authtoken
3. 复制你的 Authtoken

#### 2A.3 配置 Authtoken

```powershell
C:\tools\ngrok\ngrok.exe authtoken <你的 Authtoken>
```

#### 2A.4 启动隧道（映射 8080 端口）

```powershell
C:\tools\ngrok\ngrok.exe http 8080
```

启动后，终端会显示如下信息：

```
Forwarding  https://abc123-def.ngrok-free.app -> http://localhost:8080
```

> ✅ 记住这个 HTTPS 地址：`https://abc123-def.ngrok-free.app`
> （每次重启 ngrok 都会变，所以需要每次都更新小程序代码）

> ⚠️ 常见坑：
> - ngrok 免费域名（`.ngrok-free.app`）**未备案**，无法在微信小程序后台添加为合法域名。因此只适合「不校验合法域名」的临时调试模式。
> - 如果需要体验版/正式版测试，必须使用已备案域名（cpolar 付费版、花生壳、或自有服务器）。

---

### 方案 B：使用 cpolar（国内访问更快）

#### 2B.1 下载 cpolar

访问 https://www.cpolar.com/ 下载 Windows 客户端并安装。

#### 2B.2 注册并登录

安装后打开 cpolar 客户端，用手机号注册并登录。

#### 2B.3 创建隧道

在 cpolar 客户端中：
1. 点击「创建隧道」
2. 协议选择 `http`
3. 本地地址填写 `8080`
4. 点击「创建」

或者使用命令行：

```bash
cpolar http 8080
```

启动后会得到一个 HTTPS 地址，例如：`https://12345678.cpolar.top`

> cpolar 免费版也是临时域名，同样存在备案问题。

---

## 第三步：替换小程序 baseUrl

### 3.1 打开文件

用 VS Code 打开：`student-mini/app.js`

### 3.2 修改 baseUrl

找到第 10 行：

```javascript
    baseUrl: 'https://你的真实域名/api/v1',
```

替换为 ngrok / cpolar 提供的 HTTPS 地址（**注意保留 `/api/v1` 后缀**）：

```javascript
    // 来源：ngrok 内网穿透，映射 localhost:8080
    // 每次重启 ngrok 后需更新此地址
    baseUrl: 'https://abc123-def.ngrok-free.app/api/v1',
```

> ⚠️ **务必把 `abc123-def.ngrok-free.app` 替换为你实际拿到的地址！**

### 3.3 确认 USE_MOCK 开关（可选）

如果你想让小程序真正请求后端接口（而不是走 mock 数据），需要同时关闭 mock。

在同一文件中，将 `USE_MOCK: true` 改为：

```javascript
    USE_MOCK: false
```

> 当前 `request.js` 的逻辑是：如果 `USE_MOCK` 为 `true`，则直接返回 mock 数据，不会发网络请求。
> 建议第一次真机测试时先保持 `USE_MOCK: true`，确认页面不再白屏；然后再关闭 mock 测试后端接口。

## 第四步：配置微信小程序（二选一）

### 方案一：临时调试（不配置合法域名，推荐首次使用）

适合课程作业临时真机测试，**不需要**在微信后台配置域名。

1. 打开**微信开发者工具**
2. 点击右上角「**详情**」按钮
3. 在「**本地设置**」标签页中，勾选以下两项：
   - ✅ 「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」
   - ✅ 「不校验请求域名」
4. 点击菜单栏「**工具**」→「**预览**」
5. 用**手机微信扫描**弹出的二维码

> ⚠️ 注意：
> - 此方式生成的「预览版」小程序可以在手机上正常访问未备案的 ngrok 域名。
> - 但「体验版」和「正式版」仍然会校验域名，此设置对它们无效。

---

### 方案二：标准流程（配置合法域名）

适合需要发布体验版/正式版时使用。

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入你的小程序后台
3. 左侧菜单「**开发**」→「**开发管理**」→「**开发设置**」
4. 找到「**服务器域名**」→「**request 合法域名**」
5. 点击「**修改**」，添加你的 ngrok / cpolar 域名（不含 `https://` 前缀）：
   ```
   abc123-def.ngrok-free.app
   ```
6. 扫码确认管理员身份
7. 等待微信审核（通常几分钟内生效）

> ⚠️ 常见坑：
> - ngrok 免费域名（`.ngrok-free.app`）**未备案**，微信大概率会拒绝添加。
> - 如果你需要走标准流程，建议使用 **cpolar 付费版自定义域名** 或 **自有已备案域名 + 服务器**。
> - 课程作业临时测试，建议直接用「方案一：不校验合法域名」。

---

## 第五步：真机测试验证

### 5.1 编译上传

在微信开发者工具中：

1. 确保 `student-mini` 是打开的项目根目录
2. 点击「**编译**」按钮，确认开发者工具内无报错
3. 点击「**预览**」或「**真机调试**」

### 5.2 手机端检查

1. 用微信扫描二维码打开小程序
2. **打开调试面板**（两种方法）：
   - 方法一：点击右上角「···」→「开发调试」→「打开调试」
   - 方法二：如果是「真机调试」模式，会自动打开 vConsole
3. 观察 vConsole 中的 **Network** 标签：
   - 确认请求地址是 `https://xxx.ngrok-free.app/api/v1/...`
   - 确认状态码为 200
   - 确认返回数据正常

### 5.3 验证接口连通性

建议按以下顺序验证：

| 顺序 | 验证项 | 预期结果 |
|------|--------|----------|
| 1 | 打开首页 | 页面正常渲染，不白屏 |
| 2 | 查看 vConsole Network | 有 `/banners` 或 `/student/dashboard` 请求（取决于 USE_MOCK 开关） |
| 3 | 关闭 Mock 后刷新 | 请求发到 ngrok 地址，返回 200 |
| 4 | 登录页提交 | 请求 `POST /api/v1/auth/login`，返回 token |
| 5 | 其他功能页 | 逐个点开，确认无白屏、无网络错误 |

## 常见问题排查

### Q1：ngrok 启动后，浏览器能访问，但小程序真机请求失败

- 检查 `app.js` 中的 `baseUrl` 是否写对了完整 HTTPS 地址（含 `/api/v1`）
- 检查开发者工具「详情」→「本地设置」中是否勾选了「不校验合法域名」
- 检查 ngrok 终端是否显示请求到达（有 `200 OK` 日志）

### Q2：后端启动成功，但 ngrok 提示 "connection refused"

- 确认后端确实监听在 `localhost:8080`
- 在浏览器中访问 `http://localhost:8080/actuator/health` 确认
- 检查是否有防火墙阻止了本地连接

### Q3：真机白屏已修复，但接口返回 401 Unauthorized

- 说明请求已经发到后端，但缺少 JWT Token
- 先调用登录接口获取 token，或检查 `request.js` 中的 `Authorization` 头是否正确拼接

### Q4：每次重启 ngrok，域名都变了，需要反复改代码

- ngrok 免费版每次重启都会分配新域名，这是正常行为
- 如需固定域名，可购买 ngrok 付费版，或使用 cpolar 付费版绑定自定义域名
- 课程作业临时测试，手动改一下 `app.js` 即可

## 附录：一键启动脚本（可选）

如果你想简化操作，可以在项目根目录创建一个批处理文件 `start-backend.bat`：

```bat
@echo off
cd /d "%~dp0"
echo [1/2] 启动 Spring Boot 后端...
start "Backend" cmd /k "mvn spring-boot:run"
echo [2/2] 启动 ngrok 内网穿透...
timeout /t 5 /nobreak >nul
start "Ngrok" cmd /k "C:\tools\ngrok\ngrok.exe http 8080"
echo 请等待后端启动完成，然后查看 ngrok 窗口中的 HTTPS 地址
echo 将该地址替换到 student-mini/app.js 的 baseUrl 中
pause
```

双击运行后，会自动打开两个命令行窗口：一个跑后端，一个跑 ngrok。
