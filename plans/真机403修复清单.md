# 真机 403 错误修复清单

## 一、根因定位（基于实际代码分析）

### 1. 接口路径不存在 ❌

`pages/index/index.js` 第 36 行请求 `/banners`：
```javascript
const bannersRes = await get('/banners')
```

**后端实际路由检索结果：**
- 全局搜索 `src/main/java` 下所有 `@RequestMapping`、`@GetMapping`、`@PostMapping`
- **未找到任何 `/api/v1/banners` 路由定义**
- 后端实际提供的学生端首页聚合接口是 `GET /api/v1/student/dashboard`

### 2. Spring Security 拦截所有未授权请求

`src/main/java/edu/ruc/platform/config/SecurityConfig.java` 第 35-46 行：
```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/actuator/health", "/swagger-ui/**", "/v3/api-docs/**", 
                     "/api/v1/auth/login", "/api/v1/auth/wechat-login").permitAll()
    .anyRequest().authenticated()
)
```

**结论：** 除登录、健康检查、Swagger 外，**所有请求都需要携带有效的 Bearer Token**。请求不存在的路径时，Spring Security 会先检查认证，未登录则返回 401/403。

### 3. ngrok 拦截已排除 ✅

`student-mini/api/request.js` 第 75 行已正确配置：
```javascript
header: {
  'Content-Type': 'application/json',
  'ngrok-skip-browser-warning': 'true',
  'Authorization': app.globalData.token ? `Bearer ${app.globalData.token}` : '',
  ...header
}
```

### 4. USE_MOCK 已关闭 ✅

`student-mini/app.js` 第 11 行：
```javascript
USE_MOCK: false
```

`request.js` 第 5-10 行也能正确读取：
```javascript
let USE_MOCK = true
try {
  USE_MOCK = getApp().globalData.USE_MOCK !== false
} catch (e) {
  USE_MOCK = true
}
```

### 5. 系统性路径不匹配

`api/home.js` 中所有路径均为前端自定义，后端不存在：

| 前端路径 | 后端是否存在 | 后端实际对应接口 |
|---------|------------|----------------|
| `/mini/home` | ❌ 不存在 | `/api/v1/student/dashboard` |
| `/mini/banners` | ❌ 不存在 | （聚合在 dashboard 中） |
| `/mini/services` | ❌ 不存在 | （聚合在 dashboard 中） |
| `/mini/todos` | ❌ 不存在 | `/api/v1/student/growth-suggestions` |
| `/mini/notices` | ❌ 不存在 | `/api/v1/student/notices` |

后端实际学生端接口（`StudentSelfController.java`）：
- `GET /api/v1/student/me` — 当前学生档案
- `GET /api/v1/student/dashboard` — 首页聚合数据（含 focusItems）
- `GET /api/v1/student/growth-suggestions` — 成长建议
- `GET /api/v1/student/notices` — 我的通知
- `GET /api/v1/student/certificates/requests` — 我的证明申请
- `GET /api/v1/student/party-progress` — 党团进度
- `GET /api/v1/student/party-progress/reminders` — 党团提醒
- `GET /api/v1/student/knowledge/recommended` — 推荐知识

## 二、直接可操作的修复步骤

### 步骤 1：修复首页数据加载（pages/index/index.js）

**文件：** `student-mini/pages/index/index.js`

**当前代码（第 33-53 行）：**
```javascript
  async loadHomeData() {
    try {
      console.log('开始加载首页数据')
      const bannersRes = await get('/banners')
      console.log('bannersRes:', bannersRes)
      const newsRes = await get('/news')
      console.log('newsRes:', newsRes)
      
      this.setData({
        banners: bannersRes.data || [],
        news: newsRes.data || []
      })
      console.log('数据加载完成')
    } catch (e) {
      console.error('加载首页数据失败', e)
      wx.showToast({
        title: '加载失败，请检查网络',
        icon: 'none'
      })
    }
  }
```

**修改为：**
```javascript
  async loadHomeData() {
    try {
      console.log('开始加载首页数据')
      // 使用后端真实存在的 /student/dashboard 聚合接口
      const dashboardRes = await get('/student/dashboard')
      console.log('dashboardRes:', dashboardRes)
      
      const dashboard = dashboardRes.data || {}
      
      this.setData({
        // banners 从 dashboard 的 notices 或 focusItems 中构造，或暂时用空数组兜底
        banners: dashboard.notices ? dashboard.notices.slice(0, 3).map(n => ({
          id: n.id,
          image: '/static/images/banner1.png',
          title: n.title
        })) : [],
        // news 从 dashboard.notices 中取
        news: (dashboard.notices || []).map(n => ({
          id: n.id,
          title: n.title,
          time: n.publishTime || n.time || ''
        }))
      })
      console.log('数据加载完成')
    } catch (e) {
      console.error('加载首页数据失败', e)
      wx.showToast({
        title: '加载失败，请检查网络',
        icon: 'none'
      })
    }
  }
```

> 说明：`/student/dashboard` 返回字段包括 `profile`、`partyProgress`、`reminders`、`notices`、`certificateRequests`、`recommendedKnowledge`、`focusItems`。如果没有 banner 数据，先用 notices 构造或留空数组。

### 步骤 3：确保登录流程获取 Token

**后端认证要求：** 除 `/api/v1/auth/login`、`/api/v1/auth/wechat-login`、`/actuator/health`、`/swagger-ui/**`、`/v3/api-docs/**` 外，**所有接口都需要 `Authorization: Bearer <token>` 请求头**。

当前 `pages/index/index.js` 的 `onLoad` 没有检查登录状态，直接请求 `/student/dashboard` 会因缺少 token 而 401/403。

**建议修改 `pages/index/index.js` 的 `onLoad`：**

```javascript
  onLoad() {
    // 检查登录状态
    if (!app.isLoggedIn()) {
      wx.redirectTo({ url: '/sub-pages/login/index' })
      return
    }
    this.setData({ userInfo: app.globalData.userInfo })
    this.initDate()
    this.loadHomeData()
  }
```

同时确认 `sub-pages/login/index.js` 的登录流程能正确保存 token：
- 登录成功后调用 `app.setLoginData(res.data.token, res.data.userInfo)`
- `request.js` 中的 `Authorization` 头会从 `app.globalData.token` 读取

### 步骤 4：验证后端接口可用性（浏览器/curl 测试）

在修改小程序代码前，先用 curl 确认后端接口正常：

```bash
# 1. 登录获取 token
LOGIN_RES=$(curl -s -X POST https://camping-penny-concert.ngrok-free.dev/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"2023100001","password":"123456"}')
echo $LOGIN_RES

# 2. 提取 token（Windows PowerShell 用以下方式）
# $TOKEN = ($LOGIN_RES | ConvertFrom-Json).data.token

# 3. 请求 dashboard（携带 token）
curl -s https://camping-penny-concert.ngrok-free.dev/api/v1/student/dashboard \
  -H "Authorization: Bearer $TOKEN" \
  -H "ngrok-skip-browser-warning: true"
```

如果 curl 能正常返回 JSON，说明后端 + ngrok 链路正常，问题只在于小程序端的路径和认证。

## 四、其他页面接口路径对照表

如需继续修复其他页面，请参考以下前后端路径映射：

| 功能 | 前端当前路径 | 后端真实路径 | 所在 Controller |
|------|------------|------------|----------------|
| 登录 | `/auth/login` | `POST /api/v1/auth/login` | `AuthController` |
| 当前用户 | `/auth/me` | `GET /api/v1/auth/me` | `AuthController` |
| 首页聚合 | `/mini/home` | `GET /api/v1/student/dashboard` | `StudentSelfController` |
| 通知列表 | `/mini/notices` | `GET /api/v1/student/notices` | `StudentSelfController` |
| 成长建议 | `/mini/todos` | `GET /api/v1/student/growth-suggestions` | `StudentSelfController` |
| 证明申请 | `/affairs` | `GET /api/v1/student/certificates/requests` | `StudentSelfController` |
| 党团进度 | `/party/my-progress` | `GET /api/v1/student/party-progress` | `StudentSelfController` |
| 党团提醒 | `/party/progress/{id}/reminders` | `GET /api/v1/student/party-progress/reminders` | `StudentSelfController` |
| 知识搜索 | `/kb/search` | `GET /api/v1/knowledge/search?keyword=xxx` | `KnowledgeController` |
| 政策列表 | `/kb/policies` | （后端有 `/api/v1/knowledge/templates`，政策列表走知识库搜索） | `KnowledgeController` |
| 通知（管理端） | `/notices` | `GET /api/v1/notices/student/{studentId}` | `NoticeController` |
| 学业分析 | `/academic/overview` | `GET /api/v1/academic/analysis/{studentId}` | `AcademicWarningController` |

> 注意：后端学生端通知有两个接口：
> - `GET /api/v1/student/notices`（无需传 studentId，自动取当前登录学生）
> - `GET /api/v1/notices/student/{studentId}`（需传 studentId）
> 
> 推荐使用 `/student/notices`，因为不需要拼接 studentId。
