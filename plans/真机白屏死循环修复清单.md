# 真机白屏死循环修复清单

## 一、根因确认（基于实际代码）

### 根因 1：登录分包缺少 index.json

**文件缺失确认：** `student-mini/sub-pages/login/` 目录下只有：
- `index.js` ✅
- `index.wxml` ✅
- `index.wxss` ✅
- **`index.json` ❌ 缺失**

微信小程序**每个页面都必须有对应的 `.json` 文件**，即使内容为空 `{}`。开发者工具的模拟器可能会自动兜底，但真机环境会严格校验。缺少 `.json` 时，分包页面注册失败，导致 `rootCompPath=sub-pages/login/index not found` 错误。

### 根因 2：未登录 → 跳转登录页 → 登录页加载失败 → 页面栈崩溃 → 白屏

`pages/index/index.js` 第 15-20 行：
```javascript
  onLoad() {
    // 检查登录状态，未登录则跳转登录页
    if (!app.isLoggedIn()) {
      wx.redirectTo({ url: '/sub-pages/login/index' })
      return
    }
```

`pages/message/index.js` 第 14-18 行：
```javascript
  onLoad() {
    if (!app.isLoggedIn()) {
      wx.redirectTo({ url: '/sub-pages/login/index' })
      return
    }
```

**死循环风险分析：**
- 用户首次打开小程序（未登录），进入首页 Tab
- `onLoad` 发现 `app.isLoggedIn()` 为 `false`
- 执行 `wx.redirectTo({ url: '/sub-pages/login/index' })`
- 但 `sub-pages/login/index` 缺少 `.json` 文件，页面注册失败
- 微信框架报错 `[addView] rootCompPath=sub-pages/login/index not found`
- 由于 `redirectTo` 已经关闭了当前页面（首页），而新页面加载失败
- 页面栈为空，呈现白屏
- 如果此时有其他逻辑再次触发 `redirectTo`，就会形成循环报错

### 根因 3：profile 页面缺少登录检查

`pages/profile/index.js` 第 9-11 行的 `onLoad` **没有登录检查**：
```javascript
  onLoad() {
    this.setData({ userInfo: app.globalData.userInfo })
  }
```

当用户从「我的」Tab 进入时，如果未登录，不会跳转登录页，页面会显示空白用户信息，体验不一致。

## 二、修复步骤

### 步骤 1：创建 login 分包的 index.json

**创建文件：** `student-mini/sub-pages/login/index.json`

**文件内容：**
```json
{
  "usingComponents": {}
}
```

> ⚠️ 即使不需要自定义组件，这个文件也不能省略。微信小程序真机会严格校验每个页面的 `.json` 配置文件。

---

### 步骤 2：修复 pages/index/index.js 的跳转逻辑

**当前代码（第 15-20 行）：**
```javascript
  onLoad() {
    // 检查登录状态，未登录则跳转登录页
    if (!app.isLoggedIn()) {
      wx.redirectTo({ url: '/sub-pages/login/index' })
      return
    }
    this.setData({ userInfo: app.globalData.userInfo })
    this.initDate()
    this.loadHomeData()
  }
```

**修改为：**
```javascript
  onLoad() {
    // 未登录时跳转到登录页（使用 navigateTo 避免关闭 Tab 页）
    if (!app.isLoggedIn()) {
      wx.navigateTo({ url: '/sub-pages/login/index' })
      return
    }
    this.setData({ userInfo: app.globalData.userInfo })
    this.initDate()
    this.loadHomeData()
  }
```

> 修改说明：
> - `wx.redirectTo` 会关闭当前页面。首页是 Tab 页，用 `redirectTo` 关闭后再返回会有问题。
> - 改为 `wx.navigateTo` 更合适：登录页以页面层叠方式打开，登录成功后 `navigateBack` 即可返回首页。
> - 或者登录成功后用 `wx.switchTab` 切回首页。

## 五、常见问题

### Q：为什么开发者工具正常，真机就白屏？

A：开发者工具的模拟器对缺失 `.json` 文件有容错机制，会自动生成空配置；但真机环境（特别是基础库 2.25.0+）会严格校验，缺少 `.json` 的页面无法注册，导致 `not found` 错误。

### Q：为什么日志显示 `wx://not-found`？

A：这是微信小程序框架的内部错误码，表示框架尝试加载页面组件时找不到路径对应的页面定义。根本原因是 `.json` 缺失导致页面注册失败。

### Q：`redirectTo` 和 `navigateTo` 有什么区别？

A：
- `wx.redirectTo`：关闭当前页面，打开新页面（页面栈中当前页被替换）
- `wx.navigateTo`：保留当前页面，打开新页面（页面栈+1）
- Tab 页（首页、消息、我的）之间应使用 `wx.switchTab`
- 从 Tab 页跳转到非 Tab 页，建议使用 `wx.navigateTo`，这样登录成功后可以返回
