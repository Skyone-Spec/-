# 团队分工与本仓库职责边界

本文档用于固化课程项目的团队分工，避免职责漂移。

## 团队分工

### 人员 1：学生端小程序负责人（核心前端）

负责：

- 学生用微信小程序
- 智能问答、政策知识库页面
- 党团流程可视化、进度追踪页面
- 信息展示、精准通知查看
- 电子证明申请、学业分析页面

产出：

- 小程序全套页面
- 小程序交互逻辑

### 人员 2：后端 + 业务逻辑负责人（核心后端）

负责：

- 服务接口与核心流程
- 用户登录、身份校验、微信登录预留
- 党团流程、审批流、节点提醒
- 智能问答匹配、关键词检索
- 成绩解析、培养方案比对、选课建议的后端逻辑预留
- 全部 API 接口开发

产出：

- 接口文档
- 后端服务

### 人员 3：PC 管理后台 + 数据负责人

负责：

- 老师 / 管理员端页面
- 通知录入、标签推送
- 审批管理、流程管理页面
- 题库 / 政策库维护页面
- Excel / Word / PDF 导入导出页面与操作流
- 数据统计查看页面

产出：

- 管理后台全套功能

### 人员 4：数据库 + 文档 + 项目整合负责人

负责：

- Kingbase 数据库设计、表结构整合
- 数据加密与安全处理方案
- 数据库设计文档、用户手册
- 代码整合、部署、演示 PPT

产出：

- 库表设计
- 全部技术文档
- PPT
- 部署结果

## 本仓库当前定位

本仓库 `student-service-platform-backend` 只承载“人员 2”范围内的工作：

- 后端接口
- 后端业务逻辑
- 鉴权与权限模型
- 审批流、提醒、知识检索等服务逻辑
- 与真实数据库对接所需的实体、表结构、迁移脚本
- 接口文档

当前已固化到仓库的后端范围：

- 认证与角色鉴权
- 学生信息、学生画像、数据范围控制
- 党团流程、节点提醒、审批流后端
- 工作记录 / 工作量留痕 / 管理统计
- 知识库、附件元数据、导入任务、导入错误明细
- 管理员操作日志与统计接口

## 本仓库明确不负责

以下内容不在本仓库当前交付范围内：

- 微信小程序页面开发
- PC 管理后台页面开发
- 前端 UI/交互实现
- 项目答辩 PPT
- 最终部署整合
- 用户手册整编

## 协作原则

- 前端负责人基于本仓库 API 进行联调。
- 后端优先保证接口语义稳定，不承担前端页面实现。
- 数据库负责人可基于本仓库 migration 与实体继续细化 Kingbase 适配。
- 若后续新增需求超出“人员 2”范围，应在团队内重新分配后再进入本仓库。
