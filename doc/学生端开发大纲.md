## 一、项目概述

### 1.1 产品定位
学生端网页版是面向学生的综合服务入口，提供政策查询、智能问答、党团流程、办事申请、学业分析等功能。

### 1.2 技术选型
| 模块 | 技术方案 |
|------|----------|
| 框架 | Vue 3 + Vite (复用现有前端项目) |
| UI 组件 | Element Plus |
| 路由 | Vue Router |
| 状态管理 | Pinia |
| HTTP 客户端 | Axios |
| 构建工具 | Vite |

### 1.3 复用与共享
- **API 层**：复用 `frontend/src/api/index.js` 中定义的接口
- **Hooks**：复用 `useAuth`、`useDataScope`、`usePagination`
- **样式/布局**：可参考管理端的响应式设计

### 3.1 登录与认证 (P0)

#### 3.1.1 登录页 `Login.vue`
```
┌─────────────────────────────────────┐
│         学院学生综合服务平台          │
│                                     │
│    学号: [____________]             │
│    密码: [____________]             │
│                                     │
│    ☑ 记住登录状态                   │
│                                     │
│         [ 登 录 ]                   │
│                                     │
│    忘记密码？联系管理员              │
└─────────────────────────────────────┘
```

**功能点：**
- 学号 + 密码登录
- JWT Token 存储（localStorage）
- 记住登录状态（可选）
- 登录失败提示（密码错误、账号停用等）

**API 调用：**
```javascript
POST /api/v1/auth/login
Body: { username, password }
Response: { token, userId, role }
```

**跳转逻辑：**
- 登录成功 → Dashboard
- 已登录访问登录页 → 重定向 Dashboard

## 七、注意事项

### 7.1 数据权限
- 学生端只能访问自己的数据
- 部分信息（身份证号、完整手机号）需脱敏显示
- 附件下载需校验权限

### 7.2 移动端适配
- 关键页面需支持响应式布局
- 操作按钮需适配触摸操作
- 分页需考虑移动端显示

### 7.3 复用策略
| 组件/资源 | 复用方式 |
|-----------|----------|
| API 接口 | 直接 import `@/api` |
| useAuth Hook | 直接 import 使用 |
| 通用组件 | 按需引入 Element Plus |
| 布局组件 | 独立学生端布局 |

### 7.4 命名规范
- 页面组件：PascalCase，如 `PolicyList.vue`
- 业务组件：PascalCase，如 `PartyTimeline.vue`
- 样式类：kebab-case，如 `.student-dashboard`
