## 一、项目概述

### 1.1 产品定位
面向信息学院约1000名本科生及研究生的"一站式"校园综合服务小程序，实现：
- 事务办理电子化（党团流程、证明开具等）
- 政策咨询智能化（知识库检索、FAQ）
- 信息推送精准化（按标签/年级/班级）
- 学业情况分析与预警

### 1.2 技术选型
| 模块 | 技术方案 |
|------|----------|
| 框架 | **微信小程序**（原生开发） |
| 前端技术 | Vue 3 + Vite |
| UI 组件库 | **Vant Weapp** |
| 状态管理 | Pinia / 小程序Storage |
| HTTP 客户端 | wx.request |
| 开发工具 | 微信开发者工具 |

### 1.3 视觉规范
| 属性 | 设计规范 |
|------|----------|
| 主色调 | #1677ff（蚂蚁蓝） |
| 背景色 | #ffffff（白色） |
| 文字色 | #333 / #666 / #999（灰阶） |
| 状态色 | 成功 #52c41a / 警告 #faad14 / 错误 #ff4d4f |
| 圆角 | 8-12px |
| 字体 | 无衬线字体 |

│                                 │
│  真实姓名: [____________]        │
│  学号/工号: [____________]       │

│                                 │

**交互要求：**
- 微信登录优先，支持手机号快捷登录

- 失败提示：显示剩余尝试次数
- 连续5次失败锁定30分钟

## 十、注意事项

### 10.1 微信限制
- 主包大小限制 2MB
- 单个分包 2MB
- 页面栈最多 10 层
- request 必须 HTTPS

### 10.2 防抖要求
- UC04 申请提交：防抖 + Loading
- UC03 工单提交：防抖 + Loading
- 登录按钮：防抖 + 失败次数提示

### 10.3 缓存策略
- 政策知识库：本地缓存
- 用户信息：本地缓存
- 弱网时展示缓存 + 提示

### 10.4 错误处理
- 401：清空Token跳转登录
- 400：提取message轻提示
- 网络错误：显示重试按钮
